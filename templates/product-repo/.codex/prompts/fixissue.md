# /fixissue

Use `$wp-product-orchestrator` to fix one issue end-to-end within current authorization.

1. Read issue, comments, labels, milestone, product docs, and release branch policy.
2. Prove the correct PR base from milestone/release branch.
3. Reproduce or identify root cause.
4. Implement the smallest enterprise-grade plugin/theme fix with `$wp-expert`.
5. Add/update tests when warranted.
6. Run focused validation and WordPress live proof for the changed boundary.
7. Commit focused changes. Push/create PR only when authorized.
8. Report proof, residual risk, and exact next permission if blocked.
